OPML Dashboard: Filtered RSS Feed Display

In response to a "user request" ;-) I added a couple of extra features to the OPML Dashboard:

  • (optional) limit on the number of items that are displayed in a the feed panels (the default is 4);

  • (optional) single word filter term applied to each feed. For example: my "corporate" feedroll, filtered on "google".

The filter was trivial to implement - if a search term is passed to the opmlDisplaymanager script, the feed that is loaded into each panel is actually a routed via a simple Yahoo filter pipe. Only feed items that contain the filter term make it out of the pipe and as far as the feed display panel.

The pipe took all of five minutes to build an integrate and is good enough for proof of concept/rapid prototyping, methinks?
