Wiki Powered Feeds and an Embedded Result...

This is what mediawiki should have done from the start...

["A Distributed Publishing Framework... (screencast)" via Quickie screencast - a distributed publishing framework...]

In many respects, this makes the wiki page self-disaggregating and embeddable at a section level, which is a Good Thing :-)

Something else came tomy notice on the disaggregation from last week - PCF Converter. Although it requires a subscription to be really useful, this service will take an online or uploaded pdf, and do this to it:

  • Convert PDF documents to Word

  • Extract tables from PDF documents to Excel

  • Extract all images from PDF documents

Not surprisingly, this is exactly the sort of thing I think we should be doing to documents in ORO (the OU "Open Repository Online" service), so I'll add it to my ORO wishlist, along with document previews using something like iPaper (along with the other document/filetype conversions that Scribd offers... ;-)

PS I'm still wating to see what becomes of oEmbed... I managed to find a Django library that handles oEmbed, along with a PHP/Zend oEmbed component proposal, but haven't found any other helper libraries so far... Does Drupal have an oEmbed handler plug-in yet, I wonder? ;-)
